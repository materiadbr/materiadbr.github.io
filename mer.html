MODELO ENTIDAD RELACION (MER)

Es una herramienta para visualizar y representar la estructura de una base de datos, mostrando las entidades, sus atributos y las relaciones entre ellas.

EJEMPLO

  1. Requisitos del Sistema

    2. Una tienda online necesita guardar información sobre:

    • Clientes: Cada cliente tiene un ID único, nombre y dirección.
    • Productos: Cada producto tiene un ID único, nombre y precio.
    • Pedidos: Un pedido se identifica por un ID, tiene una fecha y pertenece a un cliente.
    • Artículos del pedido: Cada pedido contiene varios productos, y es importante saber la cantidad de cada producto que se pidió.

  2. Diseño del Modelo ER

    3. A partir de los requisitos, identificamos los componentes principales:

    • Entidades

      • - CLIENTE

      - PRODUCTO

      - PEDIDO

    • Atributos

      • - CLIENTE: ID_Cliente (clave primaria), Nombre, Direccion.

      - PRODUCTO: ID_Producto (clave primaria), Nombre, Precio.

      - PEDIDO: ID_Pedido (clave primaria), Fecha_Pedido.

    • Relaciones

      • - CLIENTE y PEDIDO:

      Un cliente realiza varios pedidos.
      Un pedido es realizado por un solo cliente.
      Esto es una relación Uno a Muchos (1:N). La relación se llama REALIZA.

      - PEDIDO y PRODUCTO:

      Un pedido contiene varios productos.
      Un producto puede estar en varios pedidos.
      Esto es una relación Muchos a Muchos (N:M). La relación se llama CONTIENE.
      Esta relación tiene un atributo propio: Cantidad, que especifica cuántas unidades de un producto hay en un pedido.
